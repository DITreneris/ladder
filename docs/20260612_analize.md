Despite **888 game runs**, both v2 hardening tables are empty in production. Likely causes:

- Production API **silently falls back** to in-memory rate limiting when Supabase writes fail (`runs.py` exception handlers).
- Rate limits then apply **per Railway instance**, not globally.
- Session tokens may not persist across instances.

**Action:** Check Railway API logs for Supabase errors on cooldown/session writes; confirm deployed build matches repo `_cooldowns.py` / `session.py` paths.

## 7. Strategic insights

### What works

- Score trust pipeline (submit → leaderboard → `best_score`)
- Manager+ progression for engaged players
- CEO as endgame milestone (core proved)
- Registration → first run conversion (96%)
- Discoverability — 12 signups in one day proves people find the app

### What struggles

- External depth (8/23 ≥3 runs, not 6/8 ratio)
- First-run duration external (&lt;30s median)
- Burst-without-return (Mike_owi, Tsg_neidomu, LesTempsBTC)
- Return rate diluted by shallow new accounts

### Do not do (data-backed)

| Idea | Why defer |

| CEO 35→25y | Core hits 45.5y; not the bottleneck |
| Soft drain @ 20y | Energy deaths @ 14y+ = long runs |
| Widen F&F marketing | Depth ratio 35%, not 75% |
| v1.1 analytics dashboard now | Useful later; not this week's blocker |

|--------|-----------------|

| First-play onboarding (v2.1.1+) measurement at T+7 | Truth on tutorial impact |
